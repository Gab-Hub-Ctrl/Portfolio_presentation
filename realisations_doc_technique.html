Cette procédure a été rédigée en janvier 2025 dans le cadre de la formation Technicien en Assistance Informatique (TAI). Elle couvre l'intégralité de la configuration d'un réseau segmenté en VLAN avec services DNS et DHCP centralisés, simulé sur Cisco Packet Tracer 8.2.2.

Le document est conçu pour être exécutable par un tiers : chaque étape est numérotée, chaque commande est copiable, chaque résultat attendu est décrit.

La procédure suit un ordre opérationnel strict : installation de l'environnement, définition de l'adressage, configuration matérielle, puis validation par tests progressifs.

Le tableau d'adressage est le document central de toute infrastructure réseau. Il recense l'ensemble des équipements, leurs adresses IP, les VLANs et les passerelles associées.

Chaque série de tests valide une couche du réseau, du niveau le plus bas (attribution DHCP) jusqu'à la connectivité applicative complète (interface Web inter-sites).

Série 1
Attribution DHCP
Vérification que chaque poste obtient une adresse IP valide depuis le serveur centralisé.
Série 2
Connectivité locale
Ping entre postes du même VLAN. Validation de la segmentation intra-site.
Série 3
Connectivité distante
Ping vers la passerelle et vers les serveurs. Validation du routage inter-VLAN.
Série 4
Connectivité inter-sites
Ping entre postes de VLANs différents. Validation du routage entre segments.
Série 5
Interface Web
Accès HTTP depuis un poste client vers le serveur web. Test de résolution DNS fonctionnelle.

Chaque test inclut la commande exacte à exécuter, le résultat attendu et l'interprétation en cas d'échec. La procédure est reproductible sans supervision.

Une procédure technique n'est utile que si elle est exécutable par quelqu'un d'autre, sans avoir à demander de précisions. C'est le standard que je me suis fixé pour ce document.

Rédiger de la documentation technique, c'est anticiper les questions du lecteur avant qu'il les pose. Tableau d'adressage complet, commandes copiables, résultats attendus explicites, interprétation des erreurs : rien n'est laissé à l'improvisation.

Cette compétence est directement transférable à la rédaction de spécifications pour des systèmes IA : décrire le comportement attendu avec assez de précision pour qu'une machine, ou un humain, puisse valider la conformité sans ambiguïté.
